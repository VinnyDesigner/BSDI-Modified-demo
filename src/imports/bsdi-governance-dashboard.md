The design must follow real governance workflows used in BSDI, including organization management, departments, user management, GIS data access requests, metadata governance, and approval workflows.

Use a premium government platform design combining:

• Glassmorphism
• Neumorphism
• Minimal enterprise UI
• Smart dashboard analytics
• GIS-inspired widgets

Use ONLY Bahrain brand colors.

3. Role-Based Dashboards

Dashboard UI must change depending on login role.

USER (Organization User)

Main actions:

Request GIS Data Access
Request Data Download
View My Requests
Browse Metadata Catalog

Dashboard widgets:

My Data Requests
Pending Approvals
Available Services
Recent Downloads

Charts:

Request Status
Services Usage
Access Timeline

This role follows the Data Access Request workflow where users submit service requests which are validated and routed to IGA admins and data owners.

DEPARTMENT USER / REVIEWER

Actions:

Review Data Requests
Create Custom Roles
Assign Roles to Users
Manage Department Users

Widgets:

Pending Department Requests
Custom Roles Created
Users in Department
Active GIS Services

Charts:

Department Request Volume
Role Assignments
User Activity

This role supports custom role creation with area-based GIS permissions requiring approval from Data Owner and IGA Admin.

ENTITY ADMIN / ORGANIZATION OWNER

Actions:

Create Department
Rename Department
Manage Organization Users
Approve Department Requests
Transfer POC ownership

Widgets:

Organization Structure
Departments Overview
Active Users
Organization Requests

Charts:

Department activity
Organization requests
Dataset access by department

Organization creation workflow includes creating a primary POC, activating the organization, and recording audit logs.

BSDI SUPER ADMIN (IGA Admin)

Full governance authority.

Actions:

Create Organization
Approve Departments
Approve User Creation
Review Metadata
Approve Data Access
Approve Role Assignments
Deactivate Users

Widgets:

National Dataset Usage
Organizations Registered
Pending Governance Approvals
System Health

Charts:

National Data Access Trends
Organization Participation
Platform Usage Analytics

This role performs final governance review before service access is enabled.

BSDI INTERNAL DEPARTMENT (Monitoring)

Read-only oversight dashboard.

Widgets:

Audit Logs
Security Alerts
Platform Activity
User Behavior Analytics

Charts:

System activity timeline
Security events
Access requests monitoring

This role focuses on audit logs and governance monitoring across all workflows.
