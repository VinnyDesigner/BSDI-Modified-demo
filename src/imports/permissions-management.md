Page Title
Permissions Management

Page Description
Manage role-based access control for organizations, modules, and system features. Define which users can view, create, edit, delete, or approve specific resources within the platform.

Role Types

Super Admin
Full system access including platform configuration, organization management, user management, and permission control.

Organization Admin
Manages users, services, and resources within their assigned organization. Can configure organization settings and review approvals.

Manager
Operational role responsible for managing services, reviewing data, and coordinating workflows within assigned modules.

Standard User
Limited access role that can view data and perform specific assigned actions within permitted modules.

Viewer
Read-only access to dashboards, reports, and spatial data without modification rights.

Permission Levels

Full Access
Allows view, create, edit, delete, and approve actions.

Manage Access
Allows create, edit, and update actions but restricts critical system configuration.

Limited Access
Allows view and limited updates within specific modules.

Read Only
Allows viewing information without any modification privileges.
